# ШІ в RisenLab — дизайн і стан інтеграції

*Оновлено: 2026-07-17. Це живий документ: «Зроблено» — реальний код у репо, «План» — погоджений напрям.*

## 1. Покращення текстур — ЗРОБЛЕНО (чекає лише API-ключ)

### Як увімкнути (одна дія)
1. Зареєструватись на [replicate.com](https://replicate.com) → Account → **API tokens** → скопіювати токен (`r8_…`).
2. У застосунку: **Налаштування → «ШІ — покращення текстур» → API-ключ** → вставити.

Все. Кнопки «Покращити текстури» (і в Моделях, і в Анімаціях) з цього моменту йдуть через
справжній ШІ. Без ключа — все працює як раніше (локальний Lanczos-апскейл), нічого не ламається.

### Архітектура (реальний код)
- `src/ai.rs` — увесь провайдерний шар:
  - `load_config()` — читає `aiApiKey`/`aiModel` з `settings.json` застосунку
    (`Desktop\RisenLab-Project\settings.json`); env `RISENLAB_AI_KEY` перекриває ключ для
    разових CLI-запусків.
  - `enhance_png()` — повний цикл Replicate: `POST /v1/models/{owner/name}/predictions` з
    `Prefer: wait` (більшість текстур повертається одним запитом) → поллінг до 240с →
    скачування результату. HTTP: `ureq` + системний TLS (SChannel) — свідомо БЕЗ rustls
    (його `ring` вимагає C-компілятор, якого нема в дев-пісочниці).
  - `texture_prompt()` — промпт за категорією텍 текстури, виведеною з її ж імені в бібліотеці
    (Monster/Wolf/Ogre → шкіра/хутро істот; Wpn/Axe/Armor → метал і шкіра зброї; Nat_/Stone →
    камінь/кора; Arch_ → архітектура; Cloth → тканина). Кожен промпт несе незмінну вимогу:
    **ті самі кольори, розкладка, силует і межі UV** — інакше текстура не ляже на модель.
  - `is_data_map()` — normal/specular НІКОЛИ не йдуть у ШІ (це вектори, не фото; фотомоделі
    їх знищують) — завжди локальний шлях.
- `batch::regenerate(..., RegenEngine)` — рушій `auto|lanczos|ai`. `auto` (дефолт скрізь):
  ключ є і текстура «фотоподібна» → ШІ, будь-яка помилка мережі/провайдера → голосний
  warning у stderr + тихий фолбек на Lanczos (кнопка ніколи не «ламається»).
- CLI: `risenlab regenerate <out_dir> <png_rel> [scale] [auto|lanczos|ai]`.

### Моделі
- **Дефолт: `nightmareai/real-esrgan`** — точний апскейл без промпту. Безпечний для будь-якої
  текстури: розкладка і кольори гарантовано зберігаються. Рекомендований режим.
- **Опційно: будь-яка img2img-модель Replicate** (поле «Модель» у Налаштуваннях, формат
  `owner/name`) — тоді вмикаються промпти за категорією + консервативний `strength: 0.3`.
  Більше деталей — більший ризик «попливти»; результат все одно проходить через існуючий
  цикл рев'ю (оригінал/покращена поруч) перед `apply-textures`, тож зіпсовану текстуру
  просто відхиляєш.

### Потік даних (незмінний, ШІ лише замінює крок «покращити»)
```
PNG з бібліотеки → regenerate (ШІ або Lanczos) → outputDir/edited/<та сама структура>
  → рев'ю (before/after) → apply-textures → мінімальний .pXX патч → гра
```
Оригінали гри не торкаються ніколи — і це лишається так.

## 2. Покращення анімацій — ПЛАН (кнопка-стаб уже в UI)

Читання анімацій повністю розв'язано (див. `docs/formats/`): скелет (.xmac), кейфрейми
(.xmot, включно з per-channel лічильниками — виправлено 2026-07-17), CPU-скінінг. Наступна
технічна передумова для БУДЬ-ЯКОГО покращення — **запис .xmot** (той самий формат, тільки
навпаки; розкладка запису тепер відома байт-точно, тож це реалістичний крок).

Три напрями (за пріоритетом користі/складності):
1. **Згладжування + подвоєння кадрів (30→60fps).** Не потребує зовнішнього ШІ: ресемплінг
   треків (slerp між ключами) + низькочастотний фільтр на дрижання суглобів. Локально, швидко,
   безпечно. Це перший кандидат на реалізацію після .xmot-writer'а.
2. **Очищення «сміття» руху.** Виявлення і згладжування одиничних викидів у ключах
   (glitch-кадри старого мокапу) — статистика по треку, теж локально.
3. **Ретаргет сучасного мокапу (справжній ШІ).** Взяти вільні мокап-бази (Mixamo/AMASS) або
   text-to-motion моделі (MDM, MotionGPT) → ретаргет на скелети Risen (кістки вже
   іменовані і читаються). Найбільший ефект — і найбільший обсяг роботи: потрібен мапінг
   скелетів + конвертація систем координат. Провайдера/модель обираємо, коли дійдемо.

UI-стаб: кнопка «🎬 Покращити анімацію (ШІ) — скоро» в панелі програвання (Анімації) з цим
списком у тултіпі — щоб напрям було видно в самому застосунку.

## 3. Безпека і вартість
- Ключ зберігається ЛИШЕ локально в `settings.json` (він і так локальний файл користувача).
  У git він не потрапляє (репо не містить settings.json), у логах не друкується.
- Real-ESRGAN на Replicate: ~$0.002–0.01 за текстуру (секунди GPU). Повна бібліотека
  (1342 текстури) ≈ кілька доларів; але робочий сценарій — точково по моделях/акторах.
- Обмежень запитів вистачає: кнопки шлють текстури послідовно, не батчем.
